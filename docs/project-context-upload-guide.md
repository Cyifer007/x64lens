# Project Context Upload Guide

## Purpose

Use this guide to keep future ChatGPT sessions aligned with the original project contracts.

## Upload these first

Upload these files to project context or attach them at the start of a new major session:

1. `PROJECT_CONTEXT.md`
2. `PROJECT_STATE.md`
3. `docs/contracts/development-contract.md`
4. `docs/contracts/research-contract.md`
5. `docs/contracts/context-persistence-contract.md`
6. `docs/architecture.md`
7. `docs/csc-773-integration.md`
8. `docs/sprints/sprint-01-plan.md`

## Upload these when working on implementation

- `docs/cli-contract.md`
- `docs/json-schema.md`
- `docs/semantic-taxonomy.md`
- `docs/scoring-model.md`
- `docs/mitigation-model.md`
- Current `src/*.asm` files being changed.
- Current `include/*.inc` files being changed.

## Upload these when working on the paper

- `docs/benchmark-methodology.md`
- `docs/publication-plan.md`
- `docs/csc-773-integration.md`
- `paper/notes/hypotheses.md`
- `paper/notes/related-work.md`
- `paper/notes/threats-to-validity.md`
- `paper/ieee/main.tex`

## Maintenance rule

If a session changes architecture, scope, research questions, CLI behavior, output schema, testing strategy, or sprint status, update `PROJECT_CONTEXT.md` and `PROJECT_STATE.md` before the session ends.
