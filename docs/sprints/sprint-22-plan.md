# Sprint 22 Plan

## Status

Planned first research release sprint.

## Sprint goal

Publish `v0.1.0` and the submission package from the frozen, independently
reproduced evidence base.

## Planned deliverables

- [ ] Clean validated `v0.1.0` tag.
- [ ] Source, Linux x86_64 binary, checksums, version, corpus, raw benchmark, and reproduction artifacts.
- [ ] Final paper source, figures, tables, bibliography, and submission package.
- [ ] Release notes, limitations, threats to validity, and extended retrospective.
- [ ] Post-release backlog for decoder, architecture, format, gadget-style, and deployment expansion.

## Acceptance criteria

- [ ] Tag points to the exact validated commit.
- [ ] All artifact checksums verify.
- [ ] Schema, CLI, docs, paper, benchmark, and case-study evidence agree.
- [ ] Claims remain bounded to measured tasks and versions.
- [ ] No unreviewed feature enters the release.

## Post-release direction

Potential work includes full or pluggable decoding, broader addressing and
transfer families, JOP/COP/SROP, symbolic reasoning, ARM64 and other
architectures, PE/Mach-O, richer APIs, and larger operational studies.
