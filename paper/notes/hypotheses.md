# Hypotheses

## H1, performance

An assembly-first ELF64 x86_64 gadget scanning engine can reduce wall-clock runtime and memory usage compared with common Python-based gadget discovery workflows on equivalent ELF64 targets.

## H2, coverage

A pattern-based first-semester scanner can maintain comparable coverage for a deliberately bounded set of common ROP primitives, while openly excluding full instruction-decoder equivalence.

## H3, semantic value

Semantic primitive classification can provide more actionable exploitability triage than raw gadget counts because it reports what attacker-relevant capabilities exist or are missing.

## H4, enterprise utility

A dependency-light static analyzer with stable JSON output can fit CI/CD and vulnerability-management workflows more easily than heavyweight interactive reverse-engineering tools.

## Measurement warning

These are hypotheses, not claims. The paper must report measured results and threats to validity.
